# DRY Code (Don't Repeat Yourself)

Before starting to create something new:

1.  **Check for existing usage**: Verify if the functionality or pattern already exists in the codebase.
2.  **Read the READMEs**: Check the documentation of the project and the specific files/folders you are working in.
3.  **Prioritize Reusability**: We avoid recreating the same thing multiple times. Use existing abstractions and libraries whenever possible.
