# Documentation Policy

## Updating Code

When changing an important piece of code, it is **mandatory** to update the README of its directory folder.

## Missing Documentation

If there is no README in the folder:

1.  **Create a README**: It must document the folder, its entities, usages, interfaces, architecture, and technical decisions.
2.  **Back-propagate**: If we change something, we must go back and update the documentation to reflect it.

## Exploration

When exploring a file or folder, **always read the READMEs** first.

## Linking

READMEs should mandatory include references to other READMEs so the user can walkthrough the system easily.
