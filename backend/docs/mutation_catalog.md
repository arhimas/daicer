# Mutation Catalog

> "What happens to the Database when I do X?"

This catalog documents the definitive Side Effects and Logic for every Game Action.
It maps the `ActionType` to the specific:

1.  **Logic Logic**: Calculations performed.
2.  **DB Updates**: Fields modified in Strapi.
3.  **Events**: `GameEvent` logs created.

---

## 1. MOVE

**Payload:**

```json
{
  "actorId": "documentId",
  "targetPosition": { "x": 10, "y": 20, "z": 0 },
  "mode": "walk"
}
```

**Logic:**

1.  **Validation**: Checks if Actor exists and Room exists.
2.  **Collision**: Checks against `room.config` (Terrain) and other Entities.
3.  **Pathfinding**: A\* Algorithm finds optimal path.
4.  **Cost**: Verifies path length <= `actor.speed` (or Speed \* Dash).

**DB Updates:**

- `api::entity-sheet.entity-sheet`: Updates `position` to final coordinate.

**Events:**

- `ENTITY_MOVED`: Contains `from`, `to`, `path`, `cost`.

---

## 2. ATTACK

**Payload:**

```json
{
  "actorId": "documentId",
  "targetId": "documentId",
  "weaponId": "componentId (Optional)"
}
```

**Logic:**

1.  **Context**: Fetches Actor (Stats, Actions) and Target (AC, HP).
2.  **Resolver**: Uses `actor.computedActions` (derived from Inventory/Stats) to find the Weapon/Spell.
3.  **Roll**: `d20 + attackBonus`. vs `target.armorClass`.
4.  **Damage**: `diceCount * d(value) + flat`. Critical Hit = Double Dice.

**DB Updates:**

- `api::entity-sheet.entity-sheet`: Updates `hp` on Target (decremented by damage).

**Events:**

- `ATTACK_RESULT`: Roll details (Hit/Miss/Crit, Total Damage).
- `DAMAGE_DEALT`: Logged if damage > 0.
- `ENTITY_DEATH`: Logged if HP reaches 0.

## 3. MODIFY_TERRAIN

**Payload:**

```json
{
  "actorId": "documentId",
  "center": { "x": 0, "y": 0, "z": 0 },
  "radius": 2,
  "type": "Stone"
}
```

**Logic:**

1.  **Voxel Engine**: Delegates to `api::voxel-engine.voxel-engine`.
2.  **Chunking**: Calculates affected Chunks and Local Coordinates.
3.  **Persistence**: Writes `VoxelChange` records (internal to Voxel Engine).

**DB Updates:**

- `api::voxel-change.voxel-change`: Creates persistence records for terrain edits (sparse storage).

**Events:**

- `TERRAIN_MODIFIED`: Logged with modification count.

---

## 4. DROP_ITEM

**Payload:**

```json
{
  "actorId": "documentId",
  "itemComponentId": "id"
}
```

**Logic:**

1.  **Inventory Service**: `dropItem(actorId, itemId)`.
2.  **Loot Bag**: Removes item from Actor Inventory. Creates "Loot" Entity (if not implemented as direct item entity yet, or standard EntitySheet with type 'loot').

**DB Updates:**

- `api::entity-sheet.entity-sheet` (Actor): Removes item from `inventory` component list.
- `api::entity-sheet.entity-sheet` (Loot/World): Creates new Entity or updates existing loot pile.

**Events:**

- `ITEM_DROPPED`: Logs position and item ID.
